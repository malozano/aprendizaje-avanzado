    4.2. Gradient Boosting¬∂

    Gradient Boosting generaliza la idea de boosting mediante una perspectiva de optimizaci√≥n num√©rica. En lugar de ajustar pesos de ejemplos (como AdaBoost), cada nuevo modelo se ajusta al gradiente negativo de la funci√≥n de p√©rdida.

    Desarrollado por Jerome Friedman (1999-2001), Gradient Boosting es la base te√≥rica de m√©todos modernos como XGBoost, LightGBM y CatBoost.

    Marco Conceptual¬∂

    Idea clave: Ver boosting como un problema de optimizaci√≥n en el espacio de funciones.

    Queremos encontrar una funci√≥n \(F(x)\) que minimice la p√©rdida esperada:

    \[F^* = \arg\min_F \mathbb{E}_{x,y}[L(y, F(x))]\]

    Donde \(L\) es una funci√≥n de p√©rdida diferenciable.

    Aproximaci√≥n: Construir \(F\) como una suma de funciones m√°s simples:

    \[F(x) = \sum_{m=0}^{M} f_m(x)\]

    Donde cada \(f_m\) es un modelo d√©bil (t√≠picamente un √°rbol).

    Analog√≠a con Gradient Descent¬∂

    Gradient Descent (optimizaci√≥n en espacio de par√°metros): 

    Œ∏ = Œ∏‚ÇÄ  # Inicializaci√≥n
for t in range(T):
    g = ‚àáL(Œ∏)  # Gradiente respecto a par√°metros
    Œ∏ = Œ∏ - Œ∑ * g  # Actualizaci√≥n

    Gradient Boosting (optimizaci√≥n en espacio de funciones): 

    F = F‚ÇÄ  # Inicializaci√≥n
for m in range(M):
    g = ‚àáL(F)  # Gradiente respecto a predicciones
    f_m = fit_model(g)  # Ajustar modelo a gradientes
    F = F + Œ∑ * f_m  # Actualizaci√≥n

    Algoritmo General de Gradient Boosting¬∂

    Inicializaci√≥n: 

    F‚ÇÄ(x) = arg min_Œ≥ Œ£·µ¢ L(y·µ¢, Œ≥)

T√≠picamente:
- Regresi√≥n: F‚ÇÄ(x) = mean(y)
- Clasificaci√≥n binaria: F‚ÇÄ(x) = log(p/(1-p)) donde p = mean(y)

    Para m = 1 hasta M:

    Paso 1: Calcular pseudo-residuos (gradientes negativos) 

    r·µ¢‚Çò = -[‚àÇL(y·µ¢, F(x·µ¢))/‚àÇF(x·µ¢)]_{F=F_{m-1}}

Estos son los "residuos generalizados"

    Paso 2: Ajustar modelo d√©bil a los residuos 

    h‚Çò = arg min_h Œ£·µ¢ (r·µ¢‚Çò - h(x·µ¢))¬≤

T√≠picamente h‚Çò es un √°rbol de decisi√≥n poco profundo

    Paso 3: Encontrar multiplicador √≥ptimo (line search) 

    Œ≥‚Çò = arg min_Œ≥ Œ£·µ¢ L(y·µ¢, F_{m-1}(x·µ¢) + Œ≥¬∑h‚Çò(x·µ¢))

En la pr√°ctica, a menudo se fija Œ≥‚Çò = 1

    Paso 4: Actualizar el ensemble 

    F‚Çò(x) = F_{m-1}(x) + ŒΩ¬∑Œ≥‚Çò¬∑h‚Çò(x)

Donde ŒΩ ‚àà (0,1] es el learning rate

    Predicci√≥n final: 

    ≈∑ = F‚Çò(x)

    Funciones de P√©rdida Comunes¬∂

    Regresi√≥n¬∂

    1. Squared Error (L2): 

    L(y, F) = (y - F)¬≤/2

Gradiente: ‚àÇL/‚àÇF = F - y
Residuo: r = y - F  (residuo ordinario)

    2. Absolute Error (L1): 

    L(y, F) = |y - F|

Gradiente: ‚àÇL/‚àÇF = sign(F - y)
Residuo: r = sign(y - F)
M√°s robusto a outliers que L2

    3. Huber Loss: 

    L(y, F) = {
    (y - F)¬≤/2           si |y - F| ‚â§ Œ¥
    Œ¥|y - F| - Œ¥¬≤/2      si |y - F| > Œ¥
}

Combina robustez de L1 cerca de 0 con suavidad de L2

    4. Quantile Loss: 

    L(y, F) = Œ£·µ¢ œÅ_Œ±(y·µ¢ - F(x·µ¢))

Donde œÅ_Œ±(u) = u¬∑(Œ± - ùüô(u < 0))

Permite predecir cuantiles (ej: mediana con Œ±=0.5)

    Clasificaci√≥n Binaria¬∂

    Logistic Loss (Deviance): 

    y ‚àà {-1, +1}
L(y, F) = log(1 + exp(-2yF))

Gradiente: ‚àÇL/‚àÇF = -2y/(1 + exp(2yF))
Residuo: r = 2y/(1 + exp(2yF))

Predicci√≥n: P(y=1|x) = 1/(1 + exp(-2F(x)))

    Exponential Loss: 

    L(y, F) = exp(-yF)

Gradiente: ‚àÇL/‚àÇF = -y¬∑exp(-yF)

Nota: Esta es la p√©rdida que AdaBoost minimiza impl√≠citamente

    Clasificaci√≥n Multiclase¬∂

    Softmax (Multinomial Deviance): 

    K clases, entrenar K funciones F_k(x)

L(y, F) = -Œ£‚Çñ y‚Çñ¬∑log(p‚Çñ)

Donde p‚Çñ = exp(F‚Çñ)/Œ£‚±º exp(F‚±º)

    Ejemplo con Squared Loss¬∂

    Para entender mejor, veamos un ejemplo completo con squared loss: 

    import numpy as np

# Datos sint√©ticos
np.random.seed(42)
X = np.linspace(0, 10, 100).reshape(-1, 1)
y = np.sin(X).ravel() + np.random.normal(0, 0.1, 100)

# Inicializaci√≥n: F‚ÇÄ = mean(y)
F = np.full(100, y.mean())

# Par√°metros
learning_rate = 0.1
n_estimators = 100

# Boosting manual
for m in range(n_estimators):
    # Paso 1: Calcular residuos
    residuals = y - F

    # Paso 2: Ajustar √°rbol a residuos
    # (aqu√≠ simplificado, en pr√°ctica se usa un √°rbol real)
    tree = fit_tree(X, residuals)  # √Årbol de profundidad limitada

    # Paso 3: Predecir con el nuevo √°rbol
    predictions = tree.predict(X)

    # Paso 4: Actualizar F
    F = F + learning_rate * predictions

    if m % 10 == 0:
        mse = np.mean((y - F)**2)
        print(f"Iteraci√≥n {m}: MSE = {mse:.4f}")

    Learning Rate (Shrinkage)¬∂

    El learning rate \(\nu\) controla la contribuci√≥n de cada √°rbol:

    \[F_m(x) = F_{m-1}(x) + \nu \cdot h_m(x)\]

    Efecto: - ŒΩ grande (ej: 1.0): Convergencia r√°pida, riesgo de overfitting - ŒΩ peque√±o (ej: 0.01-0.1): Convergencia lenta, mejor generalizaci√≥n

    Trade-off learning rate vs n√∫mero de √°rboles: 

    ŒΩ = 1.0 ‚Üí 50 √°rboles suficientes, pero puede overfit
ŒΩ = 0.1 ‚Üí 500 √°rboles necesarios, mejor generalizaci√≥n
ŒΩ = 0.01 ‚Üí 5000 √°rboles necesarios, excelente generalizaci√≥n

    Recomendaci√≥n pr√°ctica: - Usar ŒΩ peque√±o (0.01-0.1) - Aumentar n_estimators proporcionalmente - Usar early stopping para encontrar el n√∫mero √≥ptimo

    √Årboles de Decisi√≥n en Gradient Boosting¬∂

    Caracter√≠sticas t√≠picas: - Poco profundos: max_depth = 3-8 (vs Random Forest: sin l√≠mite) - Capturam interacciones: Profundidad k ‚Üí interacciones de orden k - R√°pidos de entrenar: √Årboles peque√±os - Complementarios: Cada √°rbol captura un patr√≥n residual diferente

    Ejemplo: 

    from sklearn.tree import DecisionTreeRegressor

# √Årbol t√≠pico en Gradient Boosting
tree = DecisionTreeRegressor(
    max_depth=3,           # Poco profundo
    min_samples_split=20,  # Evitar splits en pocas muestras
    min_samples_leaf=10    # Hojas razonablemente grandes
)

    Stochastic Gradient Boosting¬∂

    Introducido por Friedman (1999), a√±ade subsample de datos en cada iteraci√≥n: 

    Para m = 1 hasta M:
    1. Muestrear fracci√≥n f de datos (sin reemplazo)
    2. Calcular residuos en subsample
    3. Entrenar √°rbol en subsample
    4. Actualizar F en todo el dataset

    Ventajas: - ‚úÖ Reduce overfitting (m√°s regularizaci√≥n) - ‚úÖ M√°s r√°pido (menos datos por √°rbol) - ‚úÖ Introduce diversidad entre √°rboles

    Subsample t√≠pico: 0.5-0.8 

    gb = GradientBoostingClassifier(
    subsample=0.8,  # Usar 80% de datos por √°rbol
    n_estimators=100,
    learning_rate=0.1
)

    Hiperpar√°metros Importantes¬∂

    Control de complejidad:¬∂

    1. n_estimators: N√∫mero de √°rboles - M√°s √°rboles ‚Üí m√°s capacidad - Usar con learning_rate bajo y early stopping - T√≠pico: 100-1000

    2. learning_rate: Tasa de aprendizaje - Controla contribuci√≥n de cada √°rbol - T√≠pico: 0.01-0.1 - Trade-off con n_estimators

    3. max_depth: Profundidad m√°xima de √°rboles - Controla interacciones capturadas - T√≠pico: 3-8 - Profundidad k ‚Üí interacciones de orden k

    4. min_samples_split / min_samples_leaf: - Previene splits en pocas muestras - Regularizaci√≥n importante - T√≠pico: 20/10

    5. subsample: Fracci√≥n de muestras por √°rbol - Stochastic GB - T√≠pico: 0.8 - Reduce overfitting

    6. max_features: Features consideradas por split - Similar a Random Forest - T√≠pico: 'sqrt' o None - A√±ade diversidad

    Early Stopping¬∂

    Detener el entrenamiento cuando el rendimiento en validaci√≥n deja de mejorar: 

    gb = GradientBoostingClassifier(
    n_estimators=1000,        # N√∫mero grande
    learning_rate=0.01,       # Learning rate bajo
    validation_fraction=0.2,  # Fracci√≥n para validaci√≥n
    n_iter_no_change=50,      # Parar si no mejora en 50 iteraciones
    tol=1e-4,                 # Tolerancia de mejora
    random_state=42
)

gb.fit(X_train, y_train)

# N√∫mero √≥ptimo de √°rboles encontrado
print(f"N√∫mero √≥ptimo de √°rboles: {gb.n_estimators_}")

    Ventajas de Gradient Boosting¬∂

    • Muy efectivo: Estado del arte en muchos problemas
    • Flexible: Cualquier funci√≥n de p√©rdida diferenciable
    • Reduce sesgo y varianza: Mejor que bagging o boosting b√°sico
    • Maneja features mixtas: Num√©ricas y categ√≥ricas
    • Robusto: Con funciones de p√©rdida apropiadas
    • Interpretable: Feature importance disponible

    Limitaciones de Gradient Boosting¬∂

    • Propenso a overfitting: Requiere tuning cuidadoso
    • No paralelizable: Entrenamiento secuencial
    • Lento: Comparado con Random Forest
    • Sensible a hiperpar√°metros: M√°s que Random Forest
    • Sensible a outliers: Con squared loss

    Prevenci√≥n de Overfitting¬∂

    Estrategias m√∫ltiples: 

    gb = GradientBoostingClassifier(
    # 1. Reduce learning rate, aumenta √°rboles
    learning_rate=0.01,
    n_estimators=1000,

    # 2. Limita complejidad de √°rboles
    max_depth=3,
    min_samples_split=20,
    min_samples_leaf=10,

    # 3. Stochastic GB
    subsample=0.8,
    max_features='sqrt',

    # 4. Early stopping
    validation_fraction=0.2,
    n_iter_no_change=50,

    random_state=42
)

    Ejemplo Completo¬∂

    from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.ensemble import GradientBoostingClassifier
import numpy as np

# Datos

    learning_rate=0.1,
    max_depth=3,
    subsample=0.8,
    random_state=42,

)

# Evaluar

print(f"Test score: {gb.score(X_test, y_test):.4f}")

# Feature importance
importances = gb.feature_importances_
top_features = np.argsort(importances)[::-1][:5]
print("\nTop 5 features:")
for i in top_features:
    print(f"Feature {i}: {importances[i]:.4f}")

# Learning curves
train_scores = []
test_scores = []
for i, (train_pred, test_pred) in enumerate(zip(
    gb.staged_predict(X_train),
    gb.staged_predict(X_test)
)):
    train_scores.append(np.mean(train_pred == y_train))
    test_scores.append(np.mean(test_pred == y_test))

# El error converge gradualmente
print(f"\nScores cada 25 √°rboles:")
for i in [24, 49, 74, 99]:
    print(f"√Årboles {i+1}: Train={train_scores[i]:.4f}, Test={test_scores[i]:.4f}")

    4.3. HistGradientBoosting (scikit-learn)¬∂

    HistGradientBoosting es la implementaci√≥n moderna de Gradient Boosting en scikit-learn, inspirada en LightGBM y disponible desde la versi√≥n 0.21.

    Innovaci√≥n principal: Usar histogramas para discretizar features continuas, acelerando enormemente la b√∫squeda de splits.

    Diferencias con GradientBoosting tradicional¬∂

    Aspecto GradientBoosting HistGradientBoosting
    B√∫squeda de splits Exacta (todos los valores) Histogramas (valores discretizados)
    Velocidad Lenta 10-100x m√°s r√°pida
    Escalabilidad Dataset mediano Dataset grande
    Missing values No soportado Soporte nativo
    Features categ√≥ricas Manual encoding Soporte experimental
    API n_estimators max_iter

    Algoritmo basado en Histogramas¬∂

    Idea: En lugar de considerar todos los posibles valores para splits, discretizar features en bins (histogramas). 

    Feature continua: [0.1, 0.3, 0.5, 0.7, 0.9, 1.2, 1.5, ...]

Tradicional: Considerar todos los valores como candidatos a split
‚Üí O(n * p) comparaciones por nivel

Histogramas: Discretizar en 255 bins
‚Üí O(255 * p) comparaciones por nivel (independiente de n!)

    Construcci√≥n de histogramas: 

    1. Para cada feature:
   - Calcular quantiles para crear bins
   - Mapear valores continuos ‚Üí √≠ndices de bins (0-254)

2. Para encontrar mejor split:
   - Solo considerar los 255 bins como candidatos
   - Acumulaci√≥n eficiente de gradientes por bin

    Ventaja masiva: Complejidad independiente del n√∫mero de ejemplos.

    Implementaci√≥n en scikit-learn¬∂

    from sklearn.ensemble import HistGradientBoostingClassifier, HistGradientBoostingRegressor

# Clasificaci√≥n
hgb = HistGradientBoostingClassifier(
    max_iter=100,              # N√∫mero de √°rboles (equivalente a n_estimators)
    learning_rate=0.1,         # Tasa de aprendizaje
    max_depth=None,            # Sin l√≠mite (usa max_leaf_nodes)
    max_leaf_nodes=31,         # M√°ximo n√∫mero de hojas (default)
    min_samples_leaf=20,       # M√≠nimo en hojas
    l2_regularization=0.0,     # Regularizaci√≥n L2
    max_bins=255,              # N√∫mero de bins (default)
    categorical_features=None, # √çndices de features categ√≥ricas
    early_stopping='auto',     # Early stopping autom√°tico
    scoring='loss',            # M√©trica para early stopping
    validation_fraction=0.1,   # Fracci√≥n para validaci√≥n
    n_iter_no_change=10,       # Parar si no mejora
    random_state=42
)

hgb.fit(X_train, y_train)

    Comparaci√≥n de Velocidad¬∂

    Hiperpar√°metros Clave¬∂

    Control de iteraciones: - max_iter: N√∫mero de boosting iterations (100-1000) - learning_rate: Shrinkage (0.01-0.1)

    Control de complejidad: - max_leaf_nodes: M√°ximo n√∫mero de hojas (31 default) - max_depth: Profundidad m√°xima (None = sin l√≠mite) - min_samples_leaf: M√≠nimo de muestras en hojas (20 default)Regularizaci√≥n:
